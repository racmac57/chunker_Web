Setup Checklist: Moving Weekly Crime Report Project to a New Machine
This document helps you replicate your automation environment on a new machine (e.g., laptop). STEP 1: FILES TO COPY
Copy these folders/files from your desktop to your new machine:
1. Project Template
   - `TimeSCRPATemplet/TimeSCRPATemplet.aprx`
2. Script Folder
   - `ScriptsTimeSCRPATemplet/`
     - `Updated Weekly Crime Report.py`
     - `test_cycle_data.csv`
3. Config Files (place with scripts or in a 'Config' folder)
   - `chart_suffixes.csv`
   - `email_recipients.csv`
   - `email_log.csv` *(optional)*
   - `_Archive/archive_log.csv` *(if used)*
4. Input Data
   - Latest `.csv` cycle data used by the script
   - Any linked file GDBs if applicable
STEP 2: REQUIRED SOFTWARE
Make sure the following are installed:
- **ArcGIS Pro** (same version as your desktop)
- Python via ArcGIS Pro (built-in environment)
- These Python packages (via Python Package Manager in ArcGIS Pro):
  - `pandas`
  - `matplotlib`
  - `openpyxl`
  - `fpdf`
STEP 3: RECONNECT IN ARCGIS PRO
1. Open `TimeSCRPATemplet.aprx` on your new machine
2. Reconnect:
   - Script Tool path -> point to `Updated Weekly Crime Report.py`
   - Toolbox parameters (6 total: Report, Email, Archive, etc.) - CSV or data paths if broken
STEP 4: TEST RUN
Run the Script Tool with:
- [X] Report Only: True
- [X] Dry Run Mode: True
Then check your output folder:
`_25_SCRPA\TIME_Based\Reports`
Should include:
- `DRY_RUN_Summary.pdf`
- `DRY_RUN_Summary_Tables.xlsx`
- CSV tables for summary/pivot
STEP 5: OPTIONAL
- Zip the entire setup for backup
- Version your scripts with dates (e.g., `Updated_Weekly_Report_20250507.py`)
You're now ready to enhance or test future updates in a safe environment.
