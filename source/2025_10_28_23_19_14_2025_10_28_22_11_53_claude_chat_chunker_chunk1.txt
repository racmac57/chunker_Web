ClaudeExportFixer is a Python CLI tool designed to normalize Claude/Claude.ai export files (JSON/ZIP) for compatibility with the osteele chat viewer. ### Key Features
- **JSON Processing**: Parse and normalize Claude conversation exports in JSON format

- **Pretty Output**: Format output JSON with indentation for readability
- **Windows Support**: Batch scripts for easy setup and execution on Windows
- **Executable Build**: Create standalone .exe files using PyInstaller

name: CI

on:
  push:
    branches: [ main, develop ]

jobs:
  test:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
        python-version: ['3.8', '3.9', '3.10', '3.11', '3.12']

    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Python ${{ matrix.python-version }}
      uses: actions/setup-python@v5
      with:
        python-version: ${{ matrix.python-version }}
    
    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt
        pip install pytest
    
    - name: Run tests
      run: |
        pytest -v
    
    - name: Test CLI execution
      run: |
        python patch_conversations.py --help || true

  lint:
    runs-on: ubuntu-latest

    - uses: actions/checkout@v4
    
    - name: Set up Python

      with:
        python-version: '3.11'

# Contributing to ClaudeExportFixer

Thank you for your interest in contributing to ClaudeExportFixer! This document provides guidelines and instructions for contributing. ## Getting Started

1. Fork the repository
2. Clone your fork:
   ```bat
   git clone https://github.com/YOUR_USERNAME/ClaudeExportFixer.git
   cd ClaudeExportFixer
   ```
3. Set up the development environment:
   ```bat
   scripts\windows\create_venv.bat
   ```

## Development Workflow

### 1. Create a Branch

Create a new branch for your feature or bug fix:
```bat
git checkout -b feature/your-feature-name
```

Branch naming conventions:
- `feature/` - New features
- `bugfix/` - Bug fixes
- `docs/` - Documentation changes
- `refactor/` - Code refactoring
- `test/` - Test improvements

### 2. Make Your Changes

- Write clean, readable code
- Follow Python PEP 8 style guidelines
- Add comments for complex logic
- Update documentation as needed

### 3. Update CHANGELOG.md

Add your changes to the `[Unreleased]` section of CHANGELOG.md:

```markdown
## [Unreleased]

### Added
- New feature description

### Changed
- Modified functionality description

### Fixed
- Bug fix description
```

### 4. Run Tests

Ensure all tests pass before submitting:
```bat
scripts\windows\run_tests.bat
```

Add new tests for new functionality:
- Place tests in `tests/` directory
- Name test files `test_*.py`
- Use pytest conventions

### 5. Commit Your Changes

Write clear, descriptive commit messages:
```bat
git add . git commit -m "Add feature: description of changes"
```

Commit message format:
- Use present tense ("Add feature" not "Added feature")
- Be concise but descriptive
- Reference issues when applicable: "Fix #123: description"
